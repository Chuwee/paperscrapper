# PR 003: The Bot Interface

**Objective:** specific Telegram Bot implementation to handle user registration and feedback collection.

**Context:**
The core interaction loop happens here. The bot presents a paper card, and the user clicks a button. This feedback loop is what drives the future recommendation engine.

**Requirements:**

1.  **Dependencies:**
    *   Add `python-telegram-bot` to `requirements.txt`.
    *   Setup `python-dotenv` for loading the `TELEGRAM_BOT_TOKEN`.

2.  **Bot Logic (`bot.py`):**
    *   **`main()`**: Setup the `Application` and `Updater`.
    *   **`/start` Handler**:
        *   Welcome message: "Welcome to Cortex. Entering Calibration Mode."
        *   Calls `database.add_user(chat_id)`.
    *   **`send_paper(chat_id, paper_dict)`**:
        *   Constructs a formatted message (HTML/Markdown):
            ```text
            <b>Title of Paper</b>
            <i>Math.ST | 2023-10-15</i>
            
            Key Abstract: ... (first 300 chars) ...
            
            [Read Full PDF](link)
            ```
        *   **Inline Keyboard**: Two buttons in a row `[ ✅ Interested ]  [ ❌ Skip ]`.
        *   Callback data should contain the `paper_id` and the action (e.g., `like_{id}`, `dislike_{id}`).

3.  **Interaction Handler:**
    *   **Callback Query Handler**:
        *   Parses the callback data (`action`, `paper_id`).
        *   Calls `database.log_interaction(user_id, paper_id, action)`.
        *   **UX**: Edit the message text to append "Recorded: Interested" or "Recorded: Skipped" and remove the buttons (prevent double voting).

**Acceptance Criteria:**
*   Bot starts and responds to `/start`.
*   `send_paper` function correctly renders the card with buttons.
*   Clicking a button updates the DB and modifies the message to Give feedback.
